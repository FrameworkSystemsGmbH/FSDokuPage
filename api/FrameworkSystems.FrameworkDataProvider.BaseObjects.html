    DataAdapters

    Klasse zur Definition von SqlDataAdaptern mit dem grafischen VS.NET-Editor. Sinnvoller Weise werden abgeleitete Klassen als Singleton-Klasse implementiert. Sie haben ein statisches Get-Property, welches ein Objekt der abgeleiteten Klasse enthält. z.B. 'public static readonly »DataAdapters« Get = new »DataAdapters«();'. Auf die SqlDataAdapter kann dann von jeder Klasse per »DataAdapters«.Get.»AdapterName« zugegriffen werden. Bei abgeleiteten Klassen ist folgendes zu beachten:

    • statisches Get-Property erstellen, welches ein Objekt der abgeleiteten Klasse enthält
    • this.InitializeComponent() muß im Konstruktor aufgerufen werden
    • SqlDataAdapter auf die zugegriffen werden soll müssen entsprechende Zugriffslevel beiten (internal oder public)

    DataCollection

    Basis-Klasse einer Sammlung von Datenbank-Objekten. Ummantelt eine Table, auf deren Datensätze über die enthaltenen DataRecords zugegriffen werden kann. Beim Instantiieren wird ein SqlDataAdapter angegeben, mit dem per Fill- und Update-Method auf die Datenbank zugeriffen werden kann.

    DataCollectionList

    Liste zu einer DataCollection. Über die Liste kann per Index auf die DataRecords der DataCollection zugegriffen werden. Die Liste dient außerdem als Basis für DataBindings zu grafischen Elementen wie ListBoxen oder DataGrids. Sie kann auch zusätzlich einen null-Eintrag enthalten, um eine Auswahl wieder auf einen leere Wert setzen zu können.

    DataRecord

    Basis-Klasse eines Datenbank-Objekts. Ummantelt eine DataRow, auf deren Spalten per Set- und Get-Methoden zugegriffen. Implementiert die Standard-Eigenschaften ID und Name. Ein DataRecord wird immer in Zusammenhang einer DataCollection benutzt und erhlt auch einen Verweis auf diese. Per Fill- und Update-Methode kann auf die Datenbank zugegriffen werden.
